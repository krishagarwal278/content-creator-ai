# Database Constraints Reference

## Projects Table Constraints

Based on the Supabase database schema, here are the exact constraints:

### Content Type Check Constraint

**Constraint Name:** `projects_content_type_check`

**SQL:** `CHECK (content_type IN ('reel', 'short', 'vfx_movie', 'presentation'))`

**Allowed Values (case-sensitive, lowercase):**
- `reel` - Instagram/TikTok style reels
- `short` - Short-form video content
- `vfx_movie` - VFX/cinematic movie content
- `presentation` - Presentation-style content

**Error if violated:**
```
new row for relation "projects" violates check constraint "projects_content_type_check"
```

### Status Check Constraint

**Constraint Name:** `projects_status_check`

**SQL:** `CHECK (status IN ('draft', 'processing', 'completed', 'failed'))`

**Allowed Values (case-sensitive, lowercase):**
- `draft` - Project is in draft state (default)
- `processing` - Project is being processed
- `completed` - Project is finished successfully
- `failed` - Project processing failed

### Status Values

**Allowed Values:**
- `draft` - Project is in draft state
- `in_progress` - Project is being worked on
- `completed` - Project is finished
- `archived` - Project is archived

## Frontend Implementation

The form now uses dropdown selects for both `content_type` and `status` to prevent constraint violations.

### Content Type Dropdown
```tsx
<select value={formData.content_type} required>
  <option value="">Select content type...</option>
  <option value="short">Short</option>
  <option value="reel">Reel</option>
  <option value="video">Video</option>
  <option value="presentation">Presentation</option>
</select>
```

### Status Dropdown
```tsx
<select value={formData.status}>
  <option value="draft">Draft</option>
  <option value="in_progress">In Progress</option>
  <option value="completed">Completed</option>
  <option value="archived">Archived</option>
</select>
```

## Required Fields

Based on the Supabase schema:
- ✅ `name` (text) - **Required**
- ✅ `content_type` (text) - **Required** (must be one of the allowed values)
- ❌ `description` (text) - Optional
- ❌ `status` (text) - Optional (defaults to "draft")
- ❌ All other fields - Optional

## Notes

- The `id` field is auto-generated by Supabase (UUID)
- `created_at` and `updated_at` are automatically managed by Supabase
- Make sure to select a valid `content_type` from the dropdown to avoid constraint violations
